Guía de instalación del servicio

  • ¿Qué es Apache?

    Apache es un servidor web HTTP de código abierto, para plataformas Unix (BSD, GNU/Linux, etc.), Microsoft Windows, Macintosh y otras, que implementa el protocolo HTTP/1.1. es desarrollado y mantenido por una comunidad de usuarios bajo la supervisión de la Apache Software Foundation dentro del proyecto HTTP Server (httpd). Apache presenta entre otras características altamente configurables, bases de datos de autenticación y negociado de contenido, pero fue criticado por la falta de una interfaz gráfica que ayude en su configuración.

  • Historia de Apache

    Desde 1996, Apache es el servidor HTTP más usado. Jugó un papel fundamental en el desarrollo de la World Wide Web y alcanzó su máxima cuota de mercado en 2005, siendo el servidor empleado en el 70% de los sitios web en el mundo. Sin embargo, ha sufrido un descenso en su cuota de mercado en los últimos años. En 2009, se convirtió en el primer servidor web que alojó más de 100 millones de sitios web.

  • Instalación de Apache

    Nota: Solo veremos la instalación para GNU Linux.

    • Debian/Ubuntu
      $ sudo apt install apache2
    • Fedora/CentOS
      $ sudo yum install httpd
    • RedHat
      $ sudo yum install httpd
      $ sudo up2date httpd

Funciones básicas de Apache

  • Introducción

    Todos los comandos y directorios serán para las distribuciones Debian. En otras distribuciones esto podría variar. Recuerda que puedes ver el estado del servicio con:

    $ service apache2 status
    $ sudo systemctl status apache2
  • Iniciar Apache
    $ service apache2 start
    $ sudo systemctl start apache2
  • Reiniciar Apache
    • Cerrar conexiones activas
      $ service apache2 restart
      $ sudo systemctl restart apache2
    • No cerrar conexiones activas
      $ service apache2 reload
      $ sudo systemctl reload apache2
  • Parar Apache
    $ service apache2 stop
    $ sudo systemctl stop apache2

Directivas de Apache

  • Tiempo de espera
    # Gestiona el tiempo (en segundos) que tardará el servidor en enviar/recibir peticiones.
    Timeout <tiempo>
  • Peticiones simultáneas por conexión
    • Número de peticiones
      # Máximo número de peticiones simultáneas que pueden realizarse dentro de una misma conexión.
      MaxKeepAliveRequests <peticiones>
    • Tiempo de espera
      # Gestiona el tiempo (en segundos) que tardará el servidor en enviar/recibir peticiones dentro de una misma conexión.
      KeepAliveTimeout <tiempo>
  • Peticiones simultáneas cliente-servidor
    # Máximo número de peticiones simultáneas que pueden realizarse a nivel cliente-servidor.
    MaxConnectionsPerChild <conexiones>
